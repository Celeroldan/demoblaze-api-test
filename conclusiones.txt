**Conclusiones del ejercicio API DemoBlaze

**Se cumplieron todos los requisitos del ejercicio:

- Se utilizaron pruebas automatizadas con Karate para validar los servicios REST de DemoBlaze.
- Se probaron los endpoints `/signup` y `/login` con distintos casos de uso.
- Se capturaron las entradas (usuario y contraseña) y las salidas (respuesta JSON con campo `error`).
- Los resultados fueron validados con aserciones y reportados en consola y HTML.

**Observaciones técnicas:

- El endpoint `/signup` responde con `error: false` al crear un usuario nuevo, y con `error: true` si el usuario ya existe.
- El endpoint `/login` responde con `error: false` si las credenciales son correctas, y con `error: true` si son incorrectas.
- Karate permite estructurar pruebas claras, reproducibles y fáciles de mantener.
